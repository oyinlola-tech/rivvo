# Contributing Guide

Thank you for your interest in Rivvo. This project is proprietary and **not free to use**. Contributions are allowed **only with explicit written permission** from the owner.

## Before You Contribute

Please contact the owner to request permission:

**Owner:** Oluwayemi Oyinlola Michael  
**Portfolio:** `oyinlola.site`  
**Tech Firm:** `telente.site`

If approved, you will receive:

- The scope of allowed contributions.
- Branch naming and review guidelines.
- Access instructions (if needed).

## Contribution Standards

If permission is granted, follow these rules:

1. **Keep changes focused**  
   Make small, well‑scoped commits and avoid unrelated refactors.

2. **Follow code conventions**  
   Match existing formatting and conventions in the repo.

3. **Security first**  
   Do not introduce insecure dependencies or patterns.  
   Any crypto or auth changes must be reviewed carefully.

4. **Testing**  
   Where possible, add or update tests.  
   Avoid breaking existing functionality.

5. **Documentation**  
   Update docs for any changes in API, behavior, or configuration.

## Pull Request Checklist

- [ ] The change is authorized by the owner.
- [ ] The change is scoped and clearly explained.
- [ ] No secrets or private data are included.
- [ ] Docs updated (if relevant).

## Unauthorized Contributions

Unapproved contributions may be closed without review.
